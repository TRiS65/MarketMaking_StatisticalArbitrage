# Execution-Optimized Backtest

This report tests whether gross pair signals can survive an empirical maker/taker execution policy. Costs are computed from observed bid/ask fills and inferred passive fills from last-trade crossings, not from fixed 0.25x/0.50x spread haircuts.

The validation decision is deliberately separate from the out-of-sample audit. A rule that passes validation but has negative test net PnL is reported as an execution false positive, not as a tradable rule.
